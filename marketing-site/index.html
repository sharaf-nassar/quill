01 live

See your subscription burn
before it caps.

Quill polls Claude (Pro and Max plans), Codex (Plus and Pro plans), and MiniMax every three minutes and renders a row per usage-limit bucket — 5-hour, 7-day, and any model-specific caps the provider exposes. A pace marker shows exactly where you are in the current window, and the bar color shifts green → yellow → red as utilization climbs past 50%, then 80%.

  • OK Per-bucket reset countdowns — know precisely when 5 hours, 7 days, or model-specific buckets reset
  • ! Three time-visualization modes: pace marker, dual bars (elapsed vs utilization), or background fill
  • OK Workload summary rail aggregates Sessions, Projects, and Tokens across every enabled provider

02 analytics

Analytics that matter
when you're working
with an LLM.

Six things you actually need to know during an LLM session, refreshed live as the conversation moves.

→ Rate-limit awareness
Know how close you are to your subscription cap before you hit it — across every provider you have enabled.
→ Latency
Per-turn LLM response time, broken into session count, turn count, and average per turn. Spikes show up immediately.
→ Token efficiency
Tokens-per-line-of-code with a trend arrow. When the ratio degrades, you can see the prompt overhead climbing in real time.
→ Context savings
How many tokens of large content Quill kept out of your transcript via the MCP context store, and how many of those got pulled back via retrieval.
→ Code velocity
Lines changed per hour, with sparkline. Pairs with the token chart so you can see when the LLM is producing output vs spinning on revisions.
→ Routing cost
Transcript bytes injected by router/capture guidance and search snippets — the overhead a smarter context strategy adds. Quill makes that cost visible so you can tune it.

03 context

Keep big content
out of your transcript.

Web fetches, build outputs, large file reads, and bounded command results live in Quill's local MCP context store as searchable refs (source:N, chunk:N) instead of inflating the LLM transcript. The Context tab shows exactly how much you've preserved, how much got retrieved back, what router guidance cost, and the underlying telemetry — split into a closed four-category taxonomy.

  • OK Preserved — tokens written to the MCP store, with a retention subtitle (X% reused · N/M sources)
  • OK Retrieved — tokens pulled back via quill_get_context_source when the model needs exact details
  • ! Routing cost — transcript bytes injected by router/capture guidance, surfaced as a real cost (not a savings)
  • OK Telemetry — hook observation events that record session activity but neither leave nor enter the transcript

04 search

Find anything you said
to Claude or Codex.

Full-text search across every Claude Code and Codex session transcript on the machine. Tantivy index with BM25 scoring, snippet highlighting, and filters for project, host, role, date range, and git branch. Code-change pills surface which sessions touched which files.

  • OK Per-field BM25 boosts — concrete artifacts (files_modified, code_changes) outrank noisy metadata
  • OK Faceted filters: project, host, role, date range, git branch — pre-aggregated counts
  • OK Detail panel shows ±5 messages of context around every hit

05 learning

Quill notices the patterns
you keep repeating.

Two-stream LLM analysis combines tool-use observations from Claude/Codex hooks with git commit history to extract reusable rules. Each rule carries a Wilson lower-bound confidence with a 90-day half-life freshness decay, and rules above the threshold are auto-promoted to provider-scoped .md files on disk.

  • OK Wilson confidence scoring with provenance (observations, anti-patterns, project scope)
  • ! States: emerging → confirmed → stale → invalidated, surfaced as colored pills
  • OK Memory optimizer — Haiku scans CLAUDE.md/AGENTS.md + memory files, suggests delete/update/merge with undo

06 install

Get Quill.

Open source. MIT licensed. Built with Tauri + React. Three providers, three desktop platforms — pick what you need.

Providers

CC Claude Code
Auto-detected via the local Claude CLI. Quill installs hooks under ~/.claude/ on enable.
CX Codex
Talks to codex app-server. Hooks deployed under ~/.codex/ on enable.
MM MiniMax
Service-only — no local CLI. Provide the API key once; Quill polls the coding-plan endpoint.

Platforms

OK macOS
Universal DMG (x86_64 & aarch64). Notarized + signed via Apple.
OK Linux
AppImage and Debian packages. Built on Ubuntu 22.04.
OK Windows
NSIS installer. Auto-update via Tauri's updater plugin.
